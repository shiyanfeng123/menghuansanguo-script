# 战斗自动脚本集成使用说明

## 📋 概述

`CombatAutoScript` 已经集成到 `serveScript.py` 中，可以直接使用。本文档说明如何使用和配置。

## ✅ 已完成的集成

### 1. 导入模块
```python
# serveScript.py 第24行
from Kanloong_combat_script import CombatAutoScript
```

### 2. 初始化变量
```python
# serveScript.py 第199-202行（MyThread.__init__中）
self.combat_auto_instance = None
self.combat_auto_thread = None
self.combat_auto_running = False
```

### 3. 启动和停止方法
- `_start_combat_auto()` - 启动战斗自动操作
- `_stop_combat_auto()` - 停止战斗自动操作

## 🚀 使用方法

### 方法1：自动启动（推荐）

在脚本中进入战斗后自动启动：

```python
# 在 serveScript.py 的 run() 方法中，进入战斗后
if self.scriptName in ["官渡", "魔镜", "黑风", "矿产"]:
    if not self.guajiFlag:
        self._start_combat_auto()  # 已集成，直接调用
```

**当前集成位置：** `serveScript.py` 第1409行

### 方法2：手动控制

如果需要手动控制启动和停止：

```python
# 启动战斗自动操作
self._start_combat_auto()

# 停止战斗自动操作（例如：战斗结束或退出副本时）
self._stop_combat_auto()
```

## ⚙️ 配置说明

### 默认配置

在 `_start_combat_auto()` 方法中，可以修改以下配置：

```python
# 在 serveScript.py 第8857-8859行
self.combat_auto_instance.keep_support_general = False  # 是否保证辅助武将在场
self.combat_auto_instance.enable_main_heal = True      # 主角自动加血
self.combat_auto_instance.enable_main_summon = True    # 主角自动召唤
```

### 配置选项说明

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `keep_support_general` | bool | False | 是否保证辅助武将（刘备）在场，如果为True，会自动召唤刘备 |
| `enable_main_heal` | bool | True | 主角是否自动给血量低的单位加血 |
| `enable_main_summon` | bool | True | 主角是否自动召唤武将 |

### 自定义配置示例

```python
def _start_combat_auto(self):
    # ... 现有代码 ...
    if not self.combat_auto_instance:
        self.combat_auto_instance = CombatAutoScript(self)
        
        # 自定义配置
        self.combat_auto_instance.keep_support_general = True   # 保证辅助武将在场
        self.combat_auto_instance.enable_main_heal = True       # 启用加血
        self.combat_auto_instance.enable_main_summon = False    # 禁用召唤（如果不需要）
        
        # ... 其他代码 ...
```

## 🎯 多账号支持

### 自动检测账号

脚本会自动检测并处理所有账号：
- **账号0（主账号）**: `self.dm`
- **账号1（可选）**: `self.win1_dm`
- **账号2（可选）**: `self.win2_dm`

无需手动设置 `account_dm`，脚本会自动从 `self.thread` 获取！

### 多账号配置示例

如果你有多个账号，确保 `win1_dm` 和 `win2_dm` 已初始化：

```python
# 在 MyThread.__init__ 或相关方法中
self.dm = ...      # 主账号大漠对象
self.win1_dm = ... # 账号1大漠对象（如果有多开）
self.win2_dm = ... # 账号2大漠对象（如果有多开）
```

脚本会自动检测并处理所有非None的账号。

## 🔄 工作流程

### 启动流程

1. 调用 `_start_combat_auto()`
2. 创建 `CombatAutoScript` 实例（如果还没有）
3. 设置配置选项
4. 初始化战斗追踪（调用 `init_combat_tracking()`）
5. 启动战斗循环线程（调用 `run_combat_loop()`）

### 战斗循环流程

1. 检查是否是我方回合（通过检测按钮区域）
2. 如果是新回合：
   - 更新回合数
   - 重置错误计数
   - 查询技能CD状态
   - 更新单位位置和状态
3. 对每个账号执行操作：
   - 主角操作（加血、释放技能、召唤）
   - 武将操作（释放技能）
4. 等待下一轮循环

### 停止流程

1. 调用 `_stop_combat_auto()`
2. 设置 `combat_auto_running = False`
3. 清理战斗脚本资源（调用 `cleanup()`）
4. 等待线程结束

## ⚠️ 注意事项

### 1. 资源文件路径

确保所有图片资源文件存在：
- 武将图片：`serveAssets/images/auto/*.bmp`
- 技能图片：`serveAssets/images/auto/*.bmp`
- 道具图片：`serveAssets/images/auto/*.bmp`
- 按钮图片：`serveAssets/images/auto/*.bmp`

### 2. 游戏窗口大小

脚本默认配置为 **900x580** 游戏窗口。如果你的窗口大小不同，需要调整区域配置。

### 3. 战斗区域配置

所有战斗区域已在 `CombatAutoScript.__init__` 中自动配置，一般无需修改。如果需要自定义：

```python
self.combat_auto_instance.set_combat_regions(
    enemy_region=(0, 200, 500, 580),      # 敌军区域
    ally_region=(500, 200, 900, 580),      # 己方区域
    main_char_region=(680, 250, 900, 550), # 主角区域
    general_region=(480, 250, 700, 550),    # 武将区域
    turn_region=(400, 0, 500, 50),         # 回合指示器区域
    right_button_region=(400, 450, 600, 580) # 按钮区域
)
```

### 4. 线程安全

脚本已实现线程安全，可以安全地在多线程环境下使用。

### 5. 错误处理

脚本有完善的错误处理机制：
- 每个回合最多允许5次错误
- 回合操作超时（25秒）会自动跳过
- 错误不会导致脚本崩溃

## 🐛 故障排除

### 问题1：战斗自动操作不启动

**检查：**
1. 是否调用了 `_start_combat_auto()`？
2. `self.dm` 是否已初始化？
3. 是否在战斗状态下？

**解决：**
```python
# 检查大漠对象
if not self.dm:
    print("错误：大漠对象未初始化")
    return

# 检查是否已启动
if self.combat_auto_running:
    print("战斗自动操作已在运行")
else:
    self._start_combat_auto()
```

### 问题2：无法识别按钮或技能

**检查：**
1. 图片资源文件是否存在？
2. 游戏窗口大小是否为900x580？
3. 按钮区域配置是否正确？

**解决：**
- 检查 `serveAssets/images/auto/` 目录下的图片文件
- 确认游戏窗口大小
- 查看控制台输出的错误信息

### 问题3：多账号不工作

**检查：**
1. `self.win1_dm` 和 `self.win2_dm` 是否已初始化？
2. 账号数量是否正确检测？

**解决：**
```python
# 检查账号数量
print(f"检测到 {self.combat_auto_instance.get_account_count()} 个账号")

# 手动设置账号（如果需要）
# 注意：一般不需要，脚本会自动检测
```

如果需要自定义战斗逻辑，可以继承 `CombatAutoScript` 类：

```python
class MyCombatScript(CombatAutoScript):
    def main_char_action(self, account_index):
        # 自定义主角操作逻辑
        super().main_char_action(account_index)
        # 你的自定义代码
```

### 监听战斗事件

可以通过重写 `report_battle_info` 方法来监听战斗事件：

```python
# 在子类中
def report_battle_info(self, message, msg_type="info"):
    super().report_battle_info(message, msg_type)
    # 你的自定义处理
    if msg_type == "error":
        # 处理错误事件
        pass
```
