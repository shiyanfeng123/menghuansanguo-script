# serveScript.py 代码分析报告

## 📋 文件概览

- **文件大小**: 17,872 行
- **主要语言**: Python
- **用途**: 梦幻三国游戏自动化脚本
- **GUI框架**: wxPython

## 🎯 主要功能

### 1. **核心功能模块**

#### A. 游戏窗口管理 (MyThread类)
- 自动查找并绑定游戏窗口
- 支持多种窗口查找方式（通过窗口标题、子窗口枚举）
- 使用 Windows API 和 COM 接口实现后台操作
- 支持后台模式操作（无需激活窗口）

#### B. 图像识别系统
- **OCR文字识别**: 使用`color_format`配置进行彩色文字识别
- **图像匹配**: 通过图片模板匹配识别游戏元素
- **多图片路径**: 支持管道符`|`分隔的多个图片路径
- **区域识别**: 定义了多个识别区域（地图房源、左下、右下等）

#### C. 游戏脚本系统（87种脚本类型）

**副本类**:
- 官渡、黑风、魔镜
- 战魂楼、炼狱战魂楼
- 嗜血战场、英魂秘境
- 五行、溶洞、炼丹

**挂机类**:
- 龙珠、老鼠、倭寇、森罗殿
- 挂机+整点

**组合脚本**:
- 战魂+红+整点
- 战魂+红+魔镜+整点
- 日常任务流程
- 49级系列脚本

**特殊功能**:
- 整点活动
- 名将闯关
- 怪物攻城
- 西瓜保卫战
- 矿产、龙王令、引魔符等

#### D. 用户界面 (MyFrame类)
- **开始/暂停/继续/重置** 按钮
- **快捷键支持** (F5-F8)
- **设置对话框** - 配置游戏名称、队友信息、次数等
- **更新检查** - 从 Gitee 检查新版本
- **帮助说明** - 显示使用说明和注意事项

#### E. 团队协作功能
- 自动识别队友位置（团队1、团队2）
- 支持自定义队友名称
- 自动刷新地图避免队友掉线

#### F. 整点活动系统
- 自动切换到整点地图
- 识别并打击生肖怪物
- 支持多种整点模式（全打、龙+全打、走路等）
- 10个不同的整点地图配置

## ⚠️ 存在的问题

### 1. **代码结构问题**

#### A. 单一文件过大 (17,872行)
- ❌ 违反单一职责原则
- ❌ 难以维护和测试
- ❌ 难以多人协作
- ❌ 代码重用性差

#### B. 类设计不合理

- ❌ 实例变量过多（50+个）

#### B. 设计模式应用

**1. 策略模式 (Strategy Pattern)**
```python
# scripts/base_script.py
class BaseScript(ABC):
    @abstractmethod
    def execute(self):
        pass

# scripts/dungeon/guandu.py
class GuanduScript(BaseScript):
    def execute(self):
        # 官渡逻辑
        pass

# scripts/dungeon/mojing.py
class MojingScript(BaseScript):
    def execute(self):
        # 魔镜逻辑
        pass

# scripts/script_factory.py
class ScriptFactory:
    SCRIPTS = {
        "官渡": GuanduScript,
        "魔镜": MojingScript,
        # ...
    }
    
    @staticmethod
    def create(script_name):
        script_class = ScriptFactory.SCRIPTS.get(script_name)
        if script_class:
            return script_class()
        raise ValueError(f"Unknown script: {script_name}")
```

**2. 单例模式 (Singleton Pattern)**
```python
# core/window_manager.py
class WindowManager:
    _instance = None
    _lock = threading.Lock()
    
    def __new__(cls):
        if not cls._instance:
            with cls._lock:
                if not cls._instance:
                    cls._instance = super().__new__(cls)
        return cls._instance
```

**3. 工厂模式 (Factory Pattern)**
```python
# core/image_recognition.py
class ImageRecognizerFactory:
    @staticmethod
    def create_recognizer(type):
        if type == 'color':
            return ColorRecognizer()
        elif type == 'template':
            return TemplateRecognizer()
        # ...
```

```python
# utils/logger.py
import logging
from logging.handlers import RotatingFileHandler

class GameLogger:
    def __init__(self, name: str, log_file: str = 'game.log'):
        self.logger = logging.getLogger(name)
        self.logger.setLevel(logging.INFO)
        
        # 文件处理器
        file_handler = RotatingFileHandler(
            log_file, maxBytes=10*1024*1024, backupCount=5
        )
        file_handler.setFormatter(
            logging.Formatter('%(asctime)s - %(name)s - %(levelname)s - %(message)s')
        )
        
        # 控制台处理器
        console_handler = logging.StreamHandler()
        console_handler.setFormatter(
            logging.Formatter('%(levelname)s - %(message)s')
        )
        
        self.logger.addHandler(file_handler)
        self.logger.addHandler(console_handler)
    
    def info(self, msg):
        self.logger.info(msg)
    
    def error(self, msg):
        self.logger.error(msg)
    
    def warning(self, msg):
        self.logger.warning(msg)
```
