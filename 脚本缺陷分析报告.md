# Kanloong_combat_script.py 缺陷分析报告

## 🔴 **严重缺陷（Critical Issues）**

### 1. **use_revive_item 方法签名不匹配**
**位置：** 第847行
**问题：**
```python
def use_revive_item(self, dm_object, target_x, target_y):  # ❌ 缺少 account_index 参数
```
**调用位置：** 第953行、第966行已经传入了 `account_index` 参数，但方法定义中没有
**影响：** 会导致运行时 TypeError
**修复：** 将方法签名改为 `def use_revive_item(self, dm_object, target_x, target_y, account_index):`

### 2. **复活位置获取逻辑BUG（严重）**
**位置：** 第904-976行 `check_and_revive_dead_main_chars`
**问题：**
```python
# 第934行：从 unit_positions 查找阵亡主角位置
for char_name, x, y in self.unit_positions.get(i, {'main_chars': []})['main_chars']:
    if char_name == dead_char_name:
        dead_char_pos = (x, y)  # ❌ 这个列表在第411行已经被清空更新了！
        break
```
**详细：**
- 第411行：`self.unit_positions[account_index]['main_chars'] = []` - 清空列表
- 第412-416行：重新填充存活的角色
- 第934行：尝试从已清空的旧列表查找阵亡角色位置
**影响：** 无法获取阵亡主角位置，复活功能完全失效
**修复：** 在清空列表前保存阵亡角色的位置，或使用固定站位坐标

### 3. **回合数未更新**
**位置：** `auto_combat` 方法
**问题：** `self.current_turn` 从未递增，导致CD追踪完全不准确
**影响：** 所有CD检查都会认为技能/药品永远可用或永远不可用
**修复：** 在每次己方回合时递增 `self.current_turn`

### 4. **召唤功能未实现**
**位置：** 第1015-1026行 `_execute_summon`
**问题：** 只有TODO注释和打印语句，没有实际实现
**影响：** 召唤功能完全无法工作
**修复：** 实现完整的召唤流程

### 5. **use_heal_item 方法逻辑错误**
**位置：** 第886-902行
**问题：**
```python
def use_heal_item(self, dm_object):
    # ...
    self.click_ally_unit(dm_object, self.ally_region)  # ❌ ally_region 是区域元组，不是坐标
```
**影响：** `click_ally_unit` 期望接收坐标，但收到了区域元组，会导致类型错误
**修复：** 使用 `use_item_workflow` 或传递确切坐标

### 6. **召唤武将逻辑缺陷**
**位置：** 第997行
**问题：**
```python
if team_status['general_count'] >= 2:
    return  # ❌ 当有2个武将时不能召唤，但应该能替换
```
**影响：** 当已有2个武将时，无法召唤新武将替换出场时间长的武将
**修复：** 实现替换逻辑

## 🟡 **中等问题（Medium Issues）**

### 7. **技能CD检查时机问题**
**位置：** 第675行
**问题：** 即使技能使用失败（找不到按钮/图标），仍然会标记为已使用
```python
self.mark_skill_used(account_index, skill_name)  # ❌ 无条件标记
return True  # 可能实际没使用成功
```
**影响：** 可能导致技能未实际使用但被标记为CD中
**修复：** 只在确认成功使用后才标记

### 8. **图片文件未验证**
**位置：** 多处（技能、物品、按钮图片路径）
**问题：** 所有图片路径都是硬编码，启动时未验证文件是否存在
**影响：** 运行时才会发现图片缺失，难以排查问题
**修复：** 添加图片存在性检查方法

### 9. **区域坐标硬编码且未验证**
**位置：** 第461行 `bag_region = (0, 0, 500, 500)`
**问题：** 硬编码的示例坐标，可能不适用于实际游戏
**影响：** 背包检测、区域识别可能失败
**修复：** 使用实际测量的坐标，或添加坐标校准功能

### 10. **敌人位置检测可能重复**
**位置：** 第1097行
**问题：** 每回合都检测敌人位置，但不会去重
**影响：** 同一个敌人可能被多次添加到列表，导致列表越来越大
**修复：** 在更新前清空列表，或使用字典去重

### 11. **没有超时和重试机制**
**位置：** 所有点击操作
**问题：** 如果游戏卡顿或图片识别失败，可能会永久等待
**影响：** 脚本可能卡死
**修复：** 添加超时检测和失败重试机制

### 12. **线程安全问题**
**位置：** 共享状态变量
**问题：** `self.current_turn`、`self.general_tracking` 等在多线程环境下可能产生竞态条件
**影响：** 可能导致数据不一致
**修复：** 使用线程锁保护共享变量

### 13. **辅助技能目标选择不完善**
**位置：** 第841-842行
**问题：** "加血"等技能没有指定具体目标位置
```python
if self.use_skill_workflow(skill_name, dm_object, target_type=target_type, account_index=account_index):
    # ❌ 没有传递 target_x, target_y
```
**影响：** 可能点击错误的目标
**修复：** 为需要精确目标的技能传递确切坐标

### 14. **错误处理不足**
**位置：** 多处
**问题：** 缺少异常处理，如果大漠对象操作失败会直接崩溃
**影响：** 任何一个小错误都可能导致脚本停止
**修复：** 添加 try-except 块和错误恢复逻辑

### 15. **武将出战时间追踪不完整**
**位置：** 第390行 `deployed_turn`
**问题：** 记录了武将出场回合，但没有用于判断"出战时间更长"的逻辑
**影响：** 无法实现正确的武将替换逻辑
**修复：** 实现按出场回合排序和选择逻辑

### 16. **编码错误**
**位置：** 第148行
**问题：** `setelah` 应该是 `:`，可能是复制粘贴错误
```python
# {account_index setelah {'main_chars': ...}  # ❌
```
**影响：** 注释不正确，但不影响运行
**修复：** 修正为 `account_index:`
