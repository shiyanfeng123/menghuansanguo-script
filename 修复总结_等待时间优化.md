# 等待时间优化修复总结

## ✅ 已修复的问题

### 1. **面板打开后等待时间不足**
- **问题**：点击按钮后立即查找图片，面板可能还未完全打开
- **修复**：
  - 技能面板：从 0.2秒 增加到 0.5秒，并添加重试验证机制
  - 召唤面板：从 0.3秒 增加到 0.5秒，并添加面板确认机制
  - 道具面板：从 0秒 增加到 0.5秒

### 2. **图片查找缺少等待时间**
- **问题**：直接调用 `FindPic` 可能因为图片未加载完成而失败
- **修复**：
  - `check_bag_for_general`：改为使用 `find_pic_with_timeout`（2秒超时）
  - `detect_low_hp_units`：改为使用 `find_pic_with_timeout`（1秒超时）
  - `find_main_char`：改为使用 `find_pic_with_timeout`（2秒超时）
  - `check_tombstone_at_position`：改为使用 `find_pic_with_timeout`（1秒超时）
  - `_execute_summon`：改为使用 `find_pic_with_timeout`（3秒超时）

### 3. **点击操作后等待时间不足**
- **问题**：移动鼠标和点击后立即执行下一步，可能操作未生效
- **修复**：
  - 移动鼠标后：从 0.1秒 增加到 0.2秒
  - 点击按钮后：从 0.2秒 增加到 0.3秒
  - 点击技能/道具后：从 0.1秒+0.2秒 增加到 0.2秒+0.3秒
  - 点击敌人/友军后：从 0.1秒 增加到 0.2秒+0.2秒
  - 召唤武将点击后：从 0.1秒+0.3秒 增加到 0.2秒+0.5秒

### 4. **坐标转换问题**
- **问题**：部分 `FindPic` 调用未转换坐标格式（结束坐标 vs 宽度高度）
- **修复**：
  - `click_right_button`：添加坐标转换
  - `is_skill_visible`：添加坐标转换
  - `query_turn_status`：添加坐标转换
  - `check_bag_for_general`：添加坐标转换
  - `find_main_char`：添加坐标转换

### 5. **面板验证机制**
- **问题**：点击按钮后假设面板已打开，未验证
- **修复**：
  - 技能面板：添加重试验证（最多3次，每次间隔0.2秒）
  - 召唤面板：添加面板确认机制（查找武将图片验证）

### 6. **复活功能修复**
- **问题**：`use_revive_item` 方法不存在，调用会失败
- **修复**：
  - 添加 `复活药` 到 `item_images`
  - 添加 `复活药` CD配置（无CD）
  - 将 `use_revive_item` 调用改为 `use_item_workflow('复活药', ...)`
  - 复活后等待时间从 0.5秒 增加到 1.0秒

## ✅ 其他优化

1. **坐标转换统一化**：所有使用 `FindPic` 的地方都统一处理坐标格式
2. **错误处理增强**：添加了更多异常捕获和错误提示
3. **面板状态验证**：增加了面板打开状态的验证机制
4. **复活药支持**：添加了复活药的图片和CD配置

## 📝 注意事项

1. **等待时间可能需要根据实际游戏响应速度调整**
   - 如果游戏响应较慢，可以适当增加等待时间
   - 如果游戏响应较快，可以适当减少等待时间以提高效率

2. **超时时间设置**
   - 重要操作（如召唤、使用道具）使用较长超时（3秒）
   - 快速检测（如血量、墓碑）使用较短超时（1秒）

3. **面板验证机制**
   - 如果面板打开很慢，验证机制会自动重试
   - 如果3次重试都失败，会打印警告但继续执行（避免完全阻塞）
