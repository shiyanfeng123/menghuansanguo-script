# 战斗自动操作系统使用说明

## 📋 整体逻辑架构

### 核心流程

```
开始战斗
    ↓
轮到自己回合时（检测"25"）
    ↓
1️⃣ 全局检测所有账号状态
   ├─ 检测主角存活情况
   ├─ 检测武将数量和类型
   └─ 输出状态信息
    ↓
2️⃣ 主角操作
   ├─ 释放群体攻击（随机选择1-3个技能之一）
   └─ 随机使用加血道具（30%概率）
    ↓
3️⃣ 召唤判断
   ├─ 优先检查队友是否需要召唤
   │   ├─ 检查武将数量 < 2
   │   ├─ 优先召唤辅助武将（刘备）
   │   └─ 没有辅助武将则召唤输出武将
   └─ 为自己召唤武将
    ↓
4️⃣ 武将操作
   ├─ 输出武将（魔化关羽、曹操）
   │   └─ 释放群体攻击
   └─ 辅助武将（刘备）
       ├─ 检测敌军状态 → 有状态则清除
       └─ 无状态 → 使用辅助技能（加血/加速/加攻击）
    ↓完美
继续下一轮
```

## 🎯 关键功能模块

### 1. **全局状态检测**
- `get_team_status(account_index)` - 获取账号完整状态
- `has_enemy_status(account_index)` - 检测敌军是否有状态

### 2. **武将管理**
- `get_all_generals(account_index)` - 获取所有在场武将
- `has_support_general_alive(account_index)` - 检测是否有辅助武将
- `can_summon_general(account_index)` - 判断是否可以召唤

### 3. **背包检测**
- `check_bag_for_general(account_index, general_name)` - 检测背包武将
- `check_bag_for_support_general(account_index)` - 检测辅助武将

### 4. **技能操作**
- `use_skill_workflow(skill_name, dm_object, target_type)` - 完整技能流程
  - 点击技能按钮
  - 选择技能图标
  - 选择目标（敌军/友军）

### 5. **召唤策略**
- `summon_general(account_index, target_account_index)` - 召唤武将
- `check_and_summon_for_allies(current_account_index)` - 为队友召唤

## 📝 在 serveScript.py 中集成使用

### 步骤1：导入模块

在 `serveScript.py` 文件顶部添加：

```python
from Kanloong_combat_script import CombatAutoScript
```

### 步骤2：在 MyThread.__init__ 中初始化

```python
def __init__(self, scriptName):
    # ... 现有代码 ...
    
    # 初始化战斗自动操作
    self.combat_auto = None
    self.enable_combat_auto = False        # 是否启用战斗自动化
    self.keep_support_in_combat = False    # 战斗中保持辅助武将
    self.main_heal_enabled = True          # 主角加血开关
    self.main_summon_enabled = True        # 主角召唤开关
```

### 步骤3：在 MyThread.run() 中添加描绘分支

```python
def run(self):
    # ... 现有代码 ...
    
    elif self.scriptName == "战斗中自动":
        print('开始战斗中自动操作')
        
        # 初始化战斗自动操作
        self.combat_auto = CombatAutoScript(self)
        
        # 设置战斗区域（根据实际游戏界面调整坐标）
        enemy_region = (50, 200, 400, 400)      # 敌军区域
        ally_region = (900, 200, 1250, 400)     # 己方区域
        main_char_region = (900, 200, 1250, 280) # 主角区域
        general_region = (900, 280, 1250, 400)   # 武将区域
        turn_region = (600, 10, 700, 80)         # 回合指示器区域
        left_button_region = (0, 400, 300, 680)  # 左侧按钮区域
        
        self.combat_auto.set_combat_regions(
            enemy_region, ally_region, 
            main_char_region, general_region, 
            turn_region, left_button_region
        )
        
        # 设置控制开关
        self.combat_auto.keep_support_general = self.keep_support_in_combat
        self.combat_auto.enable_main_heal = self.main_heal_enabled
        self.combat_auto.enable_main_summon = self.main_summon_enabled
        
        # 执行战斗循环
        self.combat_auto.run_combat_loop()
```

## 📊 检测和决策逻辑

### 召唤决策流程

```
是否可以召唤？（武将数量 < 2）
    ↓
需要辅助武将？ && 没有辅助武将
    ↓
背包有辅助武将？
    ├─ 是 → 召唤刘备
    └─ 否 → 召唤输出武将
```

### 技能使用决策

```
辅助武将回合
    ↓
检测敌军状态
    ├─ 有状态 → 清除状态技能
    └─ 无状态 → 辅助技能（加血/加速/加攻击）or 控制技能
```

## ⚠️ 注意事项

1. **坐标精确度**：所有区域坐标必须精确测量
2. **图片质量**：确保图片清晰，建议使用BMP格式
3. **多账号同步**：确保所有账号窗口正确绑定
4. **测试建议**：先在单人模式下测试，再测试多账号
