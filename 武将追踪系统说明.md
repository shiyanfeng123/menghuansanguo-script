# 武将追踪系统说明

## 📋 核心功能

### 1. **武将追踪机制**
- 记录每个账号每个武将的上场时间
- 实时检测武将阵亡情况
- 追踪辅助武将所在的账号

### 2. **数据结构**
```python
self.general_tracking = {
    account_index: {
        general_name: {
            'type': 'support/dps',
            'deployed_turn': turn_number  # 上场回合
        }
    }
}
```

## 🎯 战斗流程

### 初始化（第一回合）
```
1. 调用 init_combat_tracking()
2. 检测所有账号的武将情况
3. 记录每个武将的上场回合
4. 标记哪个账号有辅助武将
```

### 每回合流程
```
1. self.current_turn += 1  # 回合数+1
2. 更新武将追踪：
   - 检测阵亡的武将（从追踪中移除）
   - 检测新上场的武将（记录上场回合）
   - 如果辅助武将阵亡，标记 support_general_account = None
3. 执行战斗操作
4. 召唤判断
```

### 辅助武将阵亡时的处理
```
检测到辅助武将阵亡
    ↓
1. 查找背包中有辅助武将的账号
    ↓
2. 找到该账号上场时间最长的武将
    ↓
3. 由该账号召唤刘备，替换最长武将
    ↓
4. 原辅助武将所在账号召唤输出武将
```

## 📊 具体场景

### 场景1：初始状态（第一回合）
```
账号1: 刘备、魔化关羽 (上场回合=0)
账号2: 魔化关羽、曹操 (上场回合=0)
账号3: 魔化关羽、曹操 (上场回合=0)

追踪数据:
{
    0: {'刘备': {'type': 'support', 'deployed_turn': 0},
       '魔化关羽': {'type': 'dps', 'deployed_turn': 0}},
    1: {'魔化关羽': {'type': 'dps', 'deployed_turn': 0},
       '曹操': {'type': 'dps', 'deployed_turn': 0}},
    2: {'魔化关羽': {'type': 'dps', 'deployed_turn': 0},
       '曹操': {'type': 'dps', 'deployed_turn': 0}}
}

support_general_account = 0
```

### 场景2：第六回合刘备阵亡
```
第6回合: 刘备阵亡

追踪更新:
{
    0: {'魔化关羽': {'type': 'dps', 'deployed_turn': 0}},
    ...
}

support_general_account = None

触发召唤逻辑:
1. 查找背包中有刘备的账号 → 假设是账号1
2. 账号1上场时间最长的是"魔化关羽"（第0回合上场）
3. 账号1召唤刘备替换魔化关羽
4. 账号0召唤输出武将补充位置
```

## 🔧 关键方法

### `init_combat_tracking()`
- 第一回合调用
- 初始化所有武将的追踪信息
- 记录辅助武将位置

### `update_general_tracking(account_index)`
- 每回合调用
- 检测武将阵亡和新上场
- 更新追踪数据

### `find_account_with_support_in_bag()`
- 查找背包中有辅助武将的账号
- 返回账号索引

### `get_longest_deployed_general(account_index)`
- 获取上场时间最长的武将
- 用于决定替换哪个武将

## ⚙️ 使用方法

### 在战斗开始时
```python
# 第一回合初始化
combat_auto.current_turn = 1
combat_auto.init_combat_tracking()
```

### 在每个回合
```python
# 更新回合数
combat_auto.current_turn += 1

# 更新武将追踪
for i in range(3):
    if combat_auto.account_dm[i]:
        combat_auto.update_general_tracking(i)
```

## 🚀 并行操作

三个账号的操作需要同时进行：
```python
# 使用线程池或异步操作三个账号
with ThreadPoolExecutor(max_workers=3) as executor:
    # 并行执行deputy账号的操作
    futures = []
    for i in range(3):
        if self.account_dm[i]:
            future = executor.submit(self.auto_combat, i)
            futures.append(future)
    
    # 等待所有操作完成
    for future in as_completed(futures):
        future.result()
```

## 📝 注意事项

1. **回合数管理**：确保每回合都递增 `current_turn`
2. **追踪同步**：每次武将变化都要更新追踪数据
3. **辅助武将该帐**：始终保持最新状态
4. **并行安全**：多线程访问时需要加锁保护
