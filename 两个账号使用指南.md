# 两个账号使用指南

## 📋 概述

脚本默认支持最多3个账号（主账号 + win1_dm + win2_dm），但您也可以只使用2个账号。脚本会自动检测并跳过未设置的账号（值为 `None` 的账号）。

## 🔧 配置方法

### 方式一：使用主账号 + 账号1（推荐）

如果您有两个账号，推荐使用主账号（`dm`）和第一个副账号（`win1_dm`），将 `win2_dm` 保持为 `None`：

```python
# 在 MyThread 初始化时
self.dm = CreateObject('dm.dmsoft')  # 主账号
self.win1_dm = CreateObject('dm.dmsoft')  # 账号1
self.win2_dm = None  # 账号2不设置，保持为 None

# 或者在初始化战斗脚本时，脚本会自动检测
# 如果 win2_dm 是 None，脚本会跳过账号索引 2
```

### 方式二：使用账号1 + 账号2

如果您不想使用主账号，也可以只使用两个副账号：

```python
self.dm = None  # 主账号不使用
self.win1_dm = CreateObject('dm.dmsoft')  # 账号1
self.win2_dm = CreateObject('dm.dmsoft')  # 账号2
```

**注意**：这种方式下，账号索引会发生变化：
- 账号索引 0（`dm`）= `None`（会被跳过）
- 账号索引 1（`win1_dm`）= 实际账号1
- 账号索引 2（`win2_dm`）= 实际账号2

## ⚙️ 区域配置

### 为每个账号配置战斗区域

脚本需要为**每个实际使用的账号**配置战斗区域：

```python
# 初始化战斗脚本
combat_auto = CombatAutoScript(thread_instance)

# 方式一：主账号 + 账号1 的情况
# 需要配置账号0和账号1的区域

# 账号0（主账号）的区域配置
combat_auto.set_combat_regions(
    enemy_region=(0, 0, 400, 300),     # 敌军区域
    ally_region=(400, 0, 800, 300),    # 己方区域
    main_char_region=(400, 200, 800, 400),  # 主角区域
    general_region=(400, 300, 800, 500),    # 武将区域
    turn_region=(700, 500, 850, 550),       # 回合指示器区域
    right_button_region=(1400, 400, 1920, 680),  # 右侧按钮区域
    hp_bar_regions=[...]  # 9个血量条区域
)

# 为账号0设置敌人固定点位
combat_auto.set_fixed_enemy_positions(
    account_index=0,
    enemy_positions=[
        (x1, y1),  # 敌人1
        (x2, y2),  # 敌人2
        (x3, y3),  # 敌人3
    ]
)

# 为账号1设置敌人固定点位（需要根据账号1的实际窗口坐标调整）
combat_auto.set_fixed_enemy_positions(
    account_index=1,
    enemy_positions=[
        (x1, y1),  # 敌人1（账号1窗口中的坐标）
        (x2, y2),  # 敌人2
        (x3, y3),  # 敌人3
    ]
)
```

### 重要提示

1. **区域坐标是相对于每个账号窗口的**：每个账号可能有不同的窗口位置和大小，需要分别配置
2. **敌人固定点位需要分别设置**：每个账号看到的敌人位置可能不同
3. **如果两个账号窗口大小相同**：可以复用相同的区域配置，只需要调整敌人固定点位

## 🎮 脚本运行逻辑

脚本会自动处理两个账号的情况：

1. **自动检测有效账号**：脚本会检查 `account_dm[0]`、`account_dm[1]`、`account_dm[2]`，跳过值为 `None` 的账号
2. **独立执行操作**：每个账号的操作是独立的，互不干扰
3. **全局检测执行**：回合开始时的全局检测（查询技能CD、检测状态等）会遍历所有有效账号

## 📝 完整示例

```python
# 在 serveScript.py 的 _start_combat_auto 方法中

def _start_combat_auto(self):
    """启动战斗自动操作"""
    try:
        if self.combat_auto_running:
            return
        print("启动战斗自动操作（2个账号模式）")
        self.combat_auto_running = True

        # 初始化战斗自动操作实例
        if not self.combat_auto_instance:
            self.combat_auto_instance = CombatAutoScript(self)
            
            # 方式一：配置主账号和账号1
            # 账号0（主账号）的区域配置
            self.combat_auto_instance.set_combat_regions(
                enemy_region=(0, 0, 400, 300),
                ally_region=(400, 0, 800, 300),
                main_char_region=(400, 200, 800, 400),
                general_region=(400, 300, 800, 500),
                turn_region=(700, 500, 850, 550),
                right_button_region=(1400, 400, 1920, 680),
                hp_bar_regions=[
                    # 9个血量条区域...
                ]
            )
            
            # 账号0的敌人固定点位
            self.combat_auto_instance.set_fixed_enemy_positions(
                account_index=0,
                enemy_positions=[
                    (100, 150),  # 敌人1
                    (200, 150),  # 敌人2
                    (300, 150),  # 敌人3
                ]
            )
            
            # 账号1的敌人固定点位（根据实际窗口调整）
            self.combat_auto_instance.set_fixed_enemy_positions(
                account_index=1,
                enemy_positions=[
                    (100, 150),  # 敌人1（账号1窗口中的坐标）
                    (200, 150),  # 敌人2
                    (300, 150),  # 敌人3
                ]
            )
            
            # 设置配置
            self.combat_auto_instance.keep_support_general = False
            self.combat_auto_instance.enable_main_heal = True
            
            # 启动战斗循环
            combat_thread = threading.Thread(target=self.combat_auto_instance.run_combat_loop)
            combat_thread.daemon = True
            combat_thread.start()
```

## ⚠️ 注意事项

1. **账号索引对应关系**：
   - 账号索引 0 = `self.thread.dm`（主账号）
   - 账号索引 1 = `self.thread.win1_dm`（账号1）
   - 账号索引 2 = `self.thread.win2_dm`（账号2，如果为 `None` 会被跳过）

2. **确保账号对象正确绑定**：
   ```python
   # 确保窗口已绑定到大漠对象
   if self.win1_dm:
       hwnd = self.win1_dm.FindWindow(...)  # 找到窗口句柄
       self.win1_dm.BindWindow(hwnd, ...)   # 绑定窗口
   ```

3. **测试单个账号**：如果只有一个账号，将 `win1_dm` 和 `win2_dm` 都设置为 `None`，脚本会只操作主账号

4. **区域配置一致性**：虽然每个账号可以有不同的区域配置，但建议保持一致的配置以提高稳定性

## 🔍 调试建议

1. **检查账号对象**：
   ```python
   print(f"账号0 (dm): {self.combat_auto_instance.account_dm[0]}")
   print(f"账号1 (win1_dm): {self.combat_auto_instance.account_dm[1]}")
   print(f"账号2 (win2_dm): {self.combat_auto_instance.account_dm[2]}")
   ```
   应该看到两个账号对象不为 `None`，一个账号为 `None`

2. **查看日志输出**：
   - 脚本会输出每个账号的操作日志
   - 注意查看是否有 "账号X 跳过" 的提示

3. **测试回合检测**：
   - 确保每个账号都能正确检测到己方回合
   - 检查 `is_my_turn()` 方法的返回值
